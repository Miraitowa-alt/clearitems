# 🧹 ClearItems - Fabric 1.21.x

![ClearItems Icon](src/main/resources/icon.png)

**ClearItems** 是一款针对复杂模组环境深度优化的稳定性清理模组。为了满足不同玩家对 UI 占用和视觉提醒强度的需求，我们提供了两个并行的视觉方案版本。

## ✨ 版本定位
**针对复杂模组环境深度优化的稳定性版本，将 BossBar 切换为 ActionBar 满足了不同玩家的需求。**

---

## 🌓 版本选择指南

### 🔵 v1.1.0 经典版 (BossBar 模式)
* **视觉效果**：屏幕上方常驻蓝色 **BossBar** 进度条，实时显示清理倒计时。
* **特性**：视觉冲击力强，提醒效果极佳，让玩家有充裕时间回收重要物品。
* **适用**：UI 较为空旷、需要强力提醒的服务器环境。

### 🟢 v1.1.1 稳定版 (ActionBar 模式)
* **视觉效果**：快捷栏上方 **ActionBar** 彩色文字显示。
* **稳定性优化**：针对 200+ 模组环境深度重构，彻底解决因权限判定导致的闪退问题。
* **特性**：智能变色提醒（绿/黄/红），不遮挡视线，与各种功能性 UI 模组完美兼容。
* **适用**：大型整合包、追求原版简洁风、或屏幕顶部已被其他模组 UI 占用的环境。

---

## 🚀 核心特性

* **🛡️ 智能动态白名单**：支持在游戏内实时添加或移除受保护物品。被加入白名单的物品（如钻石、红石元件等）将永远不会被自动清理。
* **📋 [新] 白名单列表查询**：(v1.1.1 专享) 使用 `list` 指令直接在游戏内查看所有受保护物品 ID。
* **📈 精准清理战报**：清理完成后，系统会精确统计并通报掉落物的**具体总个数**（例如：一堆64个石头计为64个，而非1个实体），让优化效果一目了然。
* **⚙️ 即时生效配置**：所有修改均即时生效并持久化至 `config/clearitems.json`，且具备配置自愈功能。
* **⚡ 极简性能开销**：基于原生 Fabric API 构建，代码逻辑简洁，不占额外内存或 CPU 资源。

## 🎮 指令指南

模组主指令为 `/clearitems`，支持以下子指令：

| 指令 | 说明 | 版本支持 |
| :--- | :--- | :--- |
| `set <秒>` | 修改自动清理的时间间隔 (1-3600秒) | 全版本 |
| `add <ID>` | 将指定物品加入保护名单 (支持 Tab 补全) | 全版本 |
| `remove <ID>` | 将物品从保护名单中移除 | v1.1.1 稳定性修复 |
| `list` | **[新]** 实时查看当前白名单所有物品 | **v1.1.1 专享** |
| `clear` | 立即执行手动清理并发布统计战报 | 全版本 |
| `start` / `stop` | 开启或关闭自动清理逻辑 | 全版本 |

## 🛠️ 安装要求

1.  **游戏版本**: Minecraft `1.21.x`
2.  **加载器**: [Fabric Loader](https://fabricmc.net/)
3.  **前置插件**: [Fabric API](https://modrinth.com/mod/fabric-api)

> **注意**：该模组已移除 OP 权限限制。如果你的服务器属于公网开放服务器，请注意配置文件管理或配合权限插件使用。

## 📝 配置文件示例

文件位于 `.minecraft/config/clearitems.json`：

```json
{
  "enabled": true,
  "interval": 300,
  "whiteList": [
    "minecraft:diamond",
    "minecraft:netherite_ingot",
    "minecraft:beacon"
  ]
}
